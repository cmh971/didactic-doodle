📚 Getting Started

Everything you need to go from zero to a fully-running server.

Contents 1. Add the bot 2. Run /setup 3. Use the dashboard 4. Economy basics 5. Moderation 6. Tickets 7. Hosting

1. Add the bot to your server

Use your bot's invite link (from the dashboard or the Discord Developer Portal). You'll need the Manage Server permission on the server you're adding it to. Grant it the recommended permissions so every feature works.

2. Run /setup

The fastest way to configure everything is the interactive setup wizard. Type /setup in any channel and use the arrows or dropdown to move between pages:

  • Modules — turn features on/off (economy, moderation, leveling, etc.)
  • Automod — invite blocking, spam, bad-word filter, mention limits
  • Leveling — level-up channel and auto-roles
  • Tickets — the full ticket studio
  • Welcome — greet new members with a custom card

3. Use the web dashboard

Prefer a browser? Open the dashboard, sign in with Discord (optionally Google first), and pick a server. From there you can toggle modules, edit the economy, compose announcements, design tickets, run giveaways, set up reaction roles, and watch live analytics — all without typing a single command.

4. Economy basics

Members earn and spend a per-server currency. Common commands:

  • /balance — check your wallet & bank
  • /daily, /weekly, /work — earn coins on a cooldown
  • /shop and /buy — spend coins on items
  • /gamble, /slots, /blackjack — risk it for the biscuit
  • /leaderboard — see the richest players

5. Moderation

Keep your server clean with tools like /ban, /kick, /timeout, /purge, and /warn. Warnings feed an infraction ladder you can review in the dashboard's Warnings page. Automod runs automatically once enabled in /setup.

6. Tickets

Design a support panel in /setup ▸ Tickets (or the dashboard's Tickets page), then deploy it to a channel. Members click a button (or pick a menu option) to open a private ticket. You can also enable DM tickets so members message the bot directly. Staff get Close, Claim, Lock, and Transcript controls.

7. Hosting

Sentinel is not available for self-hosting — it runs as a single managed instance so every server always gets the same secure, up-to-date version with zero setup. There's nothing to install and no server to maintain on your end.

Just invite the hosted bot, run /setup, and manage everything from the dashboard. See the command reference for the full list of what's available.
